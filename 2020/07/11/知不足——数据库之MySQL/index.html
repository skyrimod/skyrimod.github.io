MySQL

数据库是用来存储并管理数据的文件系统，MySQL只是其中的一个。在语法上，有下面四种语法：

  • DDL：增、删、改、查数据库和数据表
  • DML：增、删、改数据
  • DQL：查数据
  • DCL：授权和管理用户

约束

主键：PRIMARY KEY

主键用于约束表中的一行，在表中通过主键可以定位到准确的一行，主键不能重复记录且不能为空。

例：

默认：DEFAULT

当有默认DEFAULT约束的列，插入值为空时，将使用默认值。

例：

唯一：UNIQUE

使指定列的每一个数据值都是唯一的，禁止重复。当INSERT插入新数据和已有数据重复时，会INSERT失败。

外键：FOREING KEY

外键经常用来关联另一张表的主键。

注意：这句话应该单独放在创建table中。

事务

事务很像多线程中的锁，用来确保数据能被正确更改。

操作：

特制：

  • 原子性：不可分割的最小操作；
  • 持久性：当提交或回滚后，数据将持久化保存；
  • 隔离性：多个事务间相互独立；
  • 一致性：事务操作前后，数据总量不变。

多表

一个项目中的数据需要存储在多个表中，在建立多表时，需要消除冗余的数据。

数据库设计范式：

  • 每一列都是不可分割的原子数据项；
  • 消除列之间的依赖；
  • 消除传递(A->C)依赖（A->B,B->C,A->C）.

多表关系：

一对多（多对一）：在多的一方建立外键，指向一的一方的主键。

多对多：借助中间表（中间表最少两列，及两张表的主键）。
